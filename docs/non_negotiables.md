# Non-Negotiables
## Ignorance Must Take Form

This document lists project constraints that MUST NOT be casually revised.

These are not preferences.  
They are currently part of the canon.

If any of them is to change, the change must be explicit, justified, and coordinated at the canonical level.

---

## 1. Canonical Cell

The canonical cell has exactly nine fields:

- `who`
- `did`
- `this`
- `when`
- `confirmed_by`
- `if_ok`
- `if_doubt`
- `if_not`
- `status`

This is non-negotiable unless the theory itself is being revised.

---

## 2. `if_doubt` is first-class

`if_doubt` is not optional, not secondary, and not reducible to `if_not`.

Doubt is a distinct epistemic regime.

Any proposal that collapses doubt into:

- rejection,
- null,
- retry,
- or silence

is currently out of bounds.

---

## 3. Ghosts are not nulls

Ghost Records are identity-bearing structured absences.

They MUST NOT be treated as:

- nulls,
- empty fields,
- generic TODOs,
- or incidental metadata.

---

## 4. Closure may be blocked for epistemic reasons

A system may be structurally prevented from concluding even when it is operationally able to emit output.

Closure is a governed right, not a stylistic choice.

---

## 5. Hallucination is structural

Hallucination is not treated here as merely "wrong text."

The canonical framing is:

> Hallucination is the illegitimate conversion of structured absence into textual presence.

Any alternative framing must remain compatible with this one.

---

## 6. Reasoning is state transition

Reasoning in this project is not primarily modeled as text generation.

It is modeled as transition between epistemic states.

Text may surface from reasoning, but it is not the primary object of governance.

---

## 7. Verification is trajectory-level

Verification is not only an output check.

The system must judge trajectories, not just final answers.

Any architecture that verifies only final text is insufficient for the core theory.

---

## 8. LLMs are proposers, never sovereigns

Large language models may:

- propose,
- summarize,
- interpret,
- translate ambiguity,
- suggest candidates.

They may not be treated as final epistemic authority.

They do not get unilateral right to:

- conclude,
- erase doubt,
- suppress ghosts,
- or authorize closure.

---

## 9. Portable ignorance across layers

A central thesis of the project is that unresolved but relevant absence must survive boundary crossings.

The project MUST NOT regress to architectures where ignorance is silently converted into:

- null,
- timeout,
- empty string,
- fallback branch,
- or fluent output.

---

## 10. Book.toml is authority

`Book.toml` is the current canonical source of:

- project identity,
- thesis,
- principle,
- cell definition,
- required concepts,
- stack layout,
- build order,
- repository roots.

Work that conflicts with `Book.toml` is presumed wrong until canon is explicitly revised.

---

## 11. The four-floor stack remains distinct

The project currently depends on a four-floor architecture:

1. semantic form
2. deterministic materialization
3. protocol of action
4. epistemic governance

These floors may be refined, but they MUST NOT be collapsed casually.

---

## 12. Structure before continuation

The project is committed to:

- write before execute,
- doubt before fabrication,
- proof before closure.

Any shortcut that gains speed by erasing structure is suspect by default.

---

## 13. Canonical terminology matters

Key terms MUST retain stable meaning across:

- book,
- specs,
- formal models,
- software,
- experiments.

If a term is used differently in different places, the project will drift and break.

The glossary wins.

---

## 14. The project is theory-first, not benchmark-first

This project is not allowed to degenerate into:

- vague agent demos,
- benchmark theater,
- or prompt tricks pretending to be theory.

Implementation matters, but only if it remains faithful to the structure.

---

## 15. Final reminder

If a proposal seems convenient but:

- erases doubt,
- hides absence,
- collapses ghosts,
- weakens closure discipline,
- or makes the LLM sovereign,

it is almost certainly incompatible with the current canon.
